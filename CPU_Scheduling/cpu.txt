| # | Algorithm                | Description                                      | Advantages                  |
|---+---------------------------+--------------------------------------------------+----------------------------|
| 1 | FCFS (First Come First    | Processes executed in the order they arrive.     | Simple and easy to         |
|   | Serve)                    |                                                  | implement. May cause long  |
|   |                           |                                                  | waiting (convoy effect).   |
|---+---------------------------+--------------------------------------------------+----------------------------|
| 2 | SJF (Shortest Job First)  | Executes the process with the smallest burst     | Minimum average waiting    |
|   |                           | time first.                                      | time. Requires knowing     |
|   |                           |                                                  | burst time; starvation     |
|   |                           |                                                  | possible.                  |
|---+---------------------------+--------------------------------------------------+----------------------------|
| 3 | SRTF (Shortest Remaining  | Preemptive version of SJF; process with least    | Better response for short  |
|   | Time First)               | remaining time executes next.                    | jobs. High overhead and    |
|   |                           |                                                  | starvation possible.       |
|---+---------------------------+--------------------------------------------------+----------------------------|
| 4 | RR (Round Robin)          | Each process gets fixed CPU time (quantum).      | Fair and suitable for      |
|   |                           | CPU switches to next after each quantum.         | time-sharing systems.      |
|   |                           |                                                  | High context switching if  |
|   |                           |                                                  | quantum is small.          |
|---+---------------------------+--------------------------------------------------+----------------------------|
| 5 | Priority (Non-Preemptive) | Highest priority process runs until completion.  | Simple to control task     |
|   |                           | Lower ones wait.                                 | importance. Lower priority |
|   |                           |                                                  | processes may starve.      |
|---+---------------------------+--------------------------------------------------+----------------------------|
| 6 | Priority (Preemptive)     | Higher priority process can interrupt a running  | Better response for        |
|   |                           | lower-priority one.                              | critical tasks. May cause  |
|   |                           |                                                  | starvation; needs aging.   |
